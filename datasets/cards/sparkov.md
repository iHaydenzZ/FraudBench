# Sparkov Dataset Card (Synthetic Fraud Transactions)

## Overview
- **Name:** sparkov
- **Source:** [Sparkov Data Generation (Kaggle)](https://www.kaggle.com/datasets/kartik2112/fraud-detection)
- **Domain:** Credit card transaction fraud detection (synthetic)

## Dataset Statistics
- **Samples:** ~1,852,394 transactions (train + test concatenated)
- **Features:** 11 (after column removal)
- **Classes:** 2 (binary classification)
- **Fraud rate:** ~0.52%

## Label Meaning
- **0:** Legitimate transaction
- **1:** Fraudulent transaction

## Feature Description
| Feature | Type | Description |
|---------|------|-------------|
| category | Categorical | Merchant category (14 unique: grocery, gas, entertainment, etc.) |
| amt | Numeric | Transaction amount |
| gender | Categorical | Cardholder gender (M/F) |
| state | Categorical | US state (50 unique) |
| zip | Numeric | ZIP code |
| lat | Numeric | Cardholder latitude |
| long | Numeric | Cardholder longitude |
| city_pop | Numeric | City population |
| unix_time | Numeric | Transaction Unix timestamp |
| merch_lat | Numeric | Merchant latitude |
| merch_long | Numeric | Merchant longitude |

## Known Issues / Leakage Risks
- **Synthetic data:** Generated by Sparkov toolkit, not real transactions — patterns may not reflect real-world fraud
- **Geographic features:** lat/long + merch_lat/merch_long could enable distance-based features
- **Temporal ordering:** unix_time could introduce temporal leakage if not handled carefully

## Preprocessing Notes
- 3 categorical features: category (14), gender (2), state (50) — manageable for one-hot encoding
- 8 numeric features — StandardScaler recommended
- No missing values
- Original train/test split is discarded — we concat and re-split with our stratified splitter for cross-dataset consistency

## Split Strategy
- **Strategy:** Concat train + test, then stratified random split
- **Train/Val/Test:** 60% / 20% / 20%

## Dropped Columns
- PII: first, last, street, cc_num, dob
- High-cardinality: merchant (~700), city (~900), job (~500)
- Non-predictive: trans_num (ID), trans_date_trans_time (raw string), Unnamed: 0 (index)
